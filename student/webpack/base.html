# webpack基本概念

# 前言

学习webpack之前我们要先记住以下几点：

  • 不要死记写法

  • 不要试图学习所有功能

  • 了解原理，但是没必要深入原理

为什么我们要使用webpack呢？ 我们做项目，都会追求快速方便的将项目按质量完成，而webpack就是做这个，让我们做项目更加轻松，方便，并且可以保证质量

# 什么是前端工程化？

工程化是系统化、模块化、规范化的过程。主要如何提高整个系统生产效率的问题。

# webpack的主要功能

webpack主要包含以下功能：

  • 编译，包括js编译、css编译

  • 文件的压缩，打包，合并，公共模块提取等

  • 图片等资源的处理如压缩，合并雪碧图等

  • Tree-shaking等优化js的工具

  • webpack-dev-serve，Eslint，热更新等帮助开发的工具。

# webpck的工作流程

我们先大致了解一下webpack的工作流程

webpack作为一个打包工具，使用loader对css、静态资源进行转码处理，使这些资源转换成我们想要的版本或者正常引入等等，然后通过plugins对输出文件进行处理比如压缩体积等等，最后完成打包。这是它的工作过程。我们的工作首先就是进行正确的配置，然后就是对代码进行优化。注意这边说的不是代码本身优化，而是webpack对代码文件的一些优化手段，和开发者平时说的代码性能优化没有关系。优化方式主要就是代码分割和体积优化，这边单页和多页应用的优化方式不太一样，主要区别在提取公共依赖和异步模块的方面，在webpack4中主要使用optimization来进行配置，之后就是一些打包速度优化相关来优化一个打包速度提升工作效率。高级点的就是需要自定义loader和plugin了。

简单来说，我们使用webpack，需要做的事情一般有，对js、css、图片等资源使用loader进行编译处理，通过插件对打包文件进行处理来完成打包。在开发过程中会使用一些热更新、dev-serve等帮组我们开发的插件。
